IT 인프라 관리 핵심 가이드 & 예제

1. 네트워크 관리

기업의 데이터 흐름을 담당하며, 안정적인 서비스 제공을 목표로 합니다.

  • **주요 키워드**: TCP/IP, 라우터, 스위치, 방화벽, VPN 등.

주요 툴: iptables (리눅스 방화벽)

# iptables 설정 전체 (예시)
# ====================================================================

# 모든 규칙 삭제 및 기본 정책을 DROP(차단)으로 설정
sudo iptables -F
sudo iptables -P INPUT DROP

# 외부에서 80번 포트(웹 서비스)와 443번 포트(HTTPS)로 들어오는 트래픽만 허용
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# 설정 저장
sudo iptables-save > /etc/sysconfig/iptables

2. 서버 및 운영체제(OS) 관리

서비스를 구동하는 서버의 성능, 안정성, 보안을 관리합니다.

  • **주요 키워드**: LVM, RAID, 셸 스크립트, Cron, 패치 관리 등.

주요 툴: LVM (디스크 용량 확장)

# LVM을 활용한 디스크 용량 확장 (예시)
# ====================================================================

# 새 디스크(/dev/sdb)를 물리적 볼륨으로 만듭니다.
sudo pvcreate /dev/sdb

# 기존 볼륨 그룹(my_vg)에 새 물리적 볼륨을 추가합니다.
sudo vgextend my_vg /dev/sdb

# 논리적 볼륨(my_lv)의 크기를 10G만큼 늘립니다.
sudo lvextend -L +10G /dev/mapper/my_vg-my_lv

# 늘어난 용량에 맞게 파일 시스템을 조정합니다.
sudo resize2fs /dev/mapper/my_vg-my_lv

3. 보안 관리

인프라 전체를 외부 위협으로부터 보호하고 내부 자산의 안전을 보장합니다.

  • **주요 키워드**: IPS/IDS, SSL/TLS, ACL, 백업 및 복구.

주요 툴: rsync (원격 서버 백업)

# rsync를 이용한 원격 백업 스크립트 (예시)
# ====================================================================

#!/bin/bash

# 백업할 대상 디렉토리 (로컬)
SOURCE_DIR="/data/www"

# 백업을 받을 원격 서버 및 디렉토리
TARGET_DIR="backupuser@192.168.1.100:/backup/www"

# rsync 실행: -a (아카이브 모드), -v (진행 상황), -z (압축 전송) 옵션 사용
rsync -avz --delete $SOURCE_DIR $TARGET_DIR

4. 데이터베이스(DB) 관리

서비스의 핵심 데이터를 저장하고, 데이터 무결성을 유지합니다.

  • **주요 키워드**: 정규화, 인덱스, 쿼리 최적화, 트랜잭션.

주요 툴: MySQL (쿼리 최적화)

-- MySQL 쿼리 최적화 예제 (예시)
-- ====================================================================

-- 쿼리 속도 향상을 위해 email 컬럼에 인덱스 생성
CREATE INDEX idx_email ON users(email);

-- 인덱스가 적용된 쿼리 (속도 향상)
SELECT * FROM users WHERE email = 'test@example.com';

5. 클라우드 인프라

물리적 서버 대신 클라우드 환경을 활용하여 인프라를 구축합니다.

  • **주요 키워드**: IaaS/PaaS/SaaS, 컨테이너, IAC(Infrastructure as Code).

주요 툴: Terraform (EC2 인스턴스 생성)

# Terraform 설정 파일 전체 (main.tf 예시)
# ====================================================================

# AWS 리전 설정
provider "aws" {
  region = "ap-northeast-2"
}

# EC2 인스턴스 생성 정의
resource "aws_instance" "web_server" {
  ami           = "ami-0c94855ba99a7065f"
  instance_type = "t2.micro"
  tags = {
    Name = "MyWebServer"
  }
}

6. 자동화 및 DevOps

개발과 운영의 경계를 허물고, 인프라 관리의 효율성을 극대화합니다.

  • **주요 키워드**: CI/CD, 모니터링, Ansible/Terraform, DevOps.

주요 툴: Docker (Nginx 컨테이너 실행)

# Docker 명령어 (예시)
# ====================================================================

# Docker 이미지 다운로드
docker pull nginx:latest

# Docker 컨테이너 실행
# -p 8080:80: 호스트 포트를 컨테이너 포트로 연결
# --name my-nginx: 컨테이너 이름 지정
docker run -d -p 8080:80 --name my-nginx nginx
