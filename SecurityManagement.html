IT 인프라 관리: 보안 관리(Security Management) 가이드

이 가이드는 IT 인프라 관리자 및 보안 담당자를 대상으로, 기업의 IT 자산을 보호하고 외부 위협에 대응하기 위한 핵심 보안 관리 방법론과 구체적인 도구 활용 사례를 제공합니다.

1. 보안 관리 관련 서적들의 핵심 주제 리스트

  • 위험 관리 (Risk Management): 보안 위협을 식별, 분석, 평가하고 이를 통제하기 위한 전략 수립.
  • 접근 제어 (Access Control): 인가된 사용자만 특정 시스템이나 데이터에 접근하도록 통제하는 절차.
  • 네트워크 보안 (Network Security): 네트워크 트래픽을 모니터링하고 방화벽, 침입 탐지/방지 시스템 등을 통해 보호.
  • 엔드포인트 보안 (Endpoint Security): 개인 PC, 노트북, 모바일 장치 등 최종 사용자 기기를 보호.
  • 취약점 관리 (Vulnerability Management): 시스템 및 애플리케이션의 보안 취약점을 찾아내고 패치를 적용하여 해결.
  • 로그 및 이벤트 관리 (Log & Event Management): 시스템 로그와 보안 이벤트를 수집, 분석하여 비정상적인 활동을 탐지.
  • 데이터 보호 (Data Protection): 민감한 데이터를 암호화하고 백업하여 무단 접근이나 손실로부터 보호.

2. 핵심 주제별 설명 및 주요 키워드

위험 관리 (Risk Management)

설명: 기업의 비즈니스 목표에 영향을 미칠 수 있는 모든 보안 위협과 취약점을 식별하고, 해당 위험의 발생 가능성과 영향을 평가하는 과정입니다. 평가된 위험을 줄이거나 수용하기 위한 통제 방안을 마련합니다.
주요 키워드: 위협(Threat), 취약점(Vulnerability), 자산(Asset), 위험 평가(Risk Assessment), 통제(Control), 보안 정책(Security Policy).

접근 제어 (Access Control)

설명: 사용자 인증(Authentication)을 통해 신원을 확인하고, 인가(Authorization)된 권한만 부여하는 과정입니다. 시스템, 데이터, 애플리케이션에 대한 접근 권한을 최소한으로 유지하는 **최소 권한 원칙(Principle of Least Privilege)**이 중요합니다.
주요 키워드: 인증(Authentication), 인가(Authorization), 최소 권한(Least Privilege), 역할 기반 접근 제어(RBAC - Role-Based Access Control), 이중 인증(2FA - Two-Factor Authentication).

네트워크 보안 (Network Security)

설명: 네트워크를 통과하는 데이터와 시스템을 보호하는 포괄적인 활동입니다. **방화벽(Firewall)**을 사용하여 불필요한 트래픽을 차단하고, **IDS/IPS**를 통해 악의적인 활동을 탐지하거나 차단합니다. **VPN**은 원격 접속 시 안전한 통신을 보장합니다.
주요 키워드: 방화벽(Firewall), 침입 탐지 시스템(IDS - Intrusion Detection System), 침입 방지 시스템(IPS - Intrusion Prevention System), VPN(Virtual Private Network), 네트워크 분리(Network Segmentation).

엔드포인트 보안 (Endpoint Security)

설명: 서버, 네트워크 장비가 아닌 PC, 노트북, 모바일 장치 등 개별 최종 사용자 기기를 보호하는 것입니다. 악성코드 방지, 데이터 암호화, 애플리케이션 통제 등을 포함하며, **EDR(Endpoint Detection and Response)** 솔루션을 통해 위협을 실시간으로 탐지하고 대응합니다.
주요 키워드: 안티바이러스(Antivirus), 안티멀웨어(Antimalware), 디스크 암호화(Disk Encryption), EDR(Endpoint Detection and Response), USB 통제(USB Control).

취약점 관리 (Vulnerability Management)

설명: 시스템 및 소프트웨어에 존재하는 보안 취약점을 정기적으로 스캔하고, 우선순위를 정해 패치(Patch)를 적용하는 반복적인 과정입니다. 자동화된 스캔 도구를 활용하여 최신 위협에 대한 취약점 정보를 지속적으로 업데이트해야 합니다.
주요 키워드: 취약점 스캔(Vulnerability Scanning), 패치 관리(Patch Management), CVE(Common Vulnerabilities and Exposures), 제로데이 공격(Zero-day Attack).

로그 및 이벤트 관리 (Log & Event Management)

설명: 서버, 네트워크 장비, 애플리케이션 등에서 발생하는 모든 로그와 이벤트를 중앙 집중식으로 수집하고 분석하는 것입니다. **SIEM(Security Information and Event Management)** 시스템을 사용하여 방대한 로그 데이터에서 비정상적인 패턴이나 보안 사고의 징후를 실시간으로 탐지합니다.
주요 키워드: 로그(Log), 이벤트(Event), SIEM(Security Information and Event Management), 상관관계 분석(Correlation Analysis), 실시간 모니터링(Real-time Monitoring), 포렌식(Forensics).

데이터 보호 (Data Protection)

설명: 저장 중이거나 전송 중인 민감한 데이터의 기밀성, 무결성, 가용성을 보장하는 활동입니다. **암호화**는 데이터를 읽을 수 없게 만들고, **백업 및 복구 전략**은 데이터 손실에 대비합니다. **DLP(Data Loss Prevention)** 솔루션은 민감한 정보가 외부로 유출되는 것을 방지합니다.
주요 키워드: 암호화(Encryption), 백업(Backup), 복구(Recovery), DLP(Data Loss Prevention), 데이터 마스킹(Data Masking).

3. 보안 관리에서 주요 사용 도구의 상황별 구체적인 사용 예제

가. 방화벽 (Firewall): FortiGate, Cisco ASA 등

방화벽은 네트워크의 보안 경계를 설정하고 제어하는 가장 기본적인 도구입니다.

  • 상황 예제 1: 외부로부터의 불필요한 서비스 포트 차단

    목표: 외부에서 내부 서버로의 불필요한 포트 접근(예: RDP 3389, SQL 1433 등)을 차단하여 공격 표면을 줄인다.

    1. 방화벽 관리자 페이지 접속: 방화벽 웹 UI 또는 CLI에 접속합니다. 2. 보안 정책(Security Policy) 생성: "Policy & Objects" > "IPv4 Policy" 메뉴로 이동하여 새로운 정책을 만듭니다. 3. 정책 설정: - Source Interface: `wan1` (외부 네트워크) - Destination Interface: `internal-network` (내부 네트워크) - Source Address: `all` (모든 외부 IP) - Destination Address: `Internal_Servers` (내부 서버 그룹) - Service: `RDP` (TCP 3389), `MSSQL` (TCP 1433) - Action: `DENY` 또는 `REJECT` - Logging: `All sessions` (모든 차단 기록을 남김) 4. 정책 적용: 설정한 정책을 최상위 규칙으로 배치하여 가장 먼저 검사되도록 합니다.
  • 상황 예제 2: 특정 서비스만 외부에서 접근 허용

    목표: 외부 클라이언트가 웹 서버(80, 443 포트)에만 접근하도록 허용한다.

    1. 정책 생성: 위와 동일한 메뉴에서 새로운 정책을 만듭니다. 2. 정책 설정: - Source Interface: `wan1` - Destination Interface: `internal-network` - Source Address: `all` - Destination Address: `Web_Server` (웹 서버 IP) - Service: `HTTP` (TCP 80), `HTTPS` (TCP 443) - Action: `ACCEPT` - Logging: `All sessions` 3. 정책 순서 조정: 이 허용 정책을 위에 있는 `DENY` 정책보다 우선순위가 낮게(아래) 배치하여, `RDP`나 `MSSQL` 같은 불필요한 트래픽은 먼저 차단되고 웹 트래픽만 허용되도록 합니다.

나. SIEM (Security Information and Event Management): Splunk, Elastic Stack 등

SIEM은 다양한 소스에서 발생하는 방대한 로그를 수집하고 분석하여 보안 위협을 탐지하는 데 사용됩니다.

  • 상황 예제 1: 비정상적인 로그인 시도 탐지

    목표: 퇴사한 직원의 계정에서 평소와 다른 IP로 로그인 시도가 반복될 경우 즉시 알림을 받는다.

    1. 로그 수집: Active Directory, VPN, 웹 애플리케이션 등에서 발생하는 인증 로그를 SIEM 서버로 실시간으로 전송합니다. 2. 대시보드 생성: SIEM 대시보드에서 `Login_Attempts`를 검색하여 시간별, 사용자별, IP별 로그인 시도 현황을 시각화합니다. 3. 상관관계 규칙(Correlation Rule) 설정: - Rule Name: `Repeated_Failed_Login` - IF: `event_type = 'login' AND result = 'failed' AND username = 'terminated_employee_A'` - THEN: `count(events) > 5 within 10 minutes from source_ip` - ACTION: `Send email alert to security_team@company.com` 4. 결과: 해당 규칙이 활성화되면, 퇴사자 계정으로 10분 내에 5번 이상의 로그인 실패가 발생할 경우 즉시 담당자에게 알림이 전송됩니다.
  • 상황 예제 2: 특정 악성 IP로부터의 트래픽 탐지

    목표: 외부 위협 인텔리전스 피드에 등록된 악성 IP와 통신하는 내부 시스템을 찾아낸다.

    1. 외부 위협 피드 연동: 최신 악성 IP 주소 목록(Threat Intelligence Feed)을 SIEM 시스템에 통합합니다. 2. 네트워크 로그 수집: 방화벽, IDS/IPS, 네트워크 장비의 트래픽 로그를 SIEM으로 전송합니다. 3. 검색 쿼리 실행: SIEM의 검색창에 다음과 같은 쿼리를 입력하여 악성 IP와 통신한 로그를 찾습니다. - `search (source_ip IN (threat_intelligence_list) OR destination_ip IN (threat_intelligence_list)) AND (event_type = 'connection')` 4. 대응: 검색 결과로 탐지된 내부 시스템의 IP를 확인하고, 해당 시스템을 네트워크에서 격리하거나 추가적인 포렌식 분석을 진행합니다.

다. EDR (Endpoint Detection and Response): CrowdStrike, Carbon Black 등

EDR은 엔드포인트(PC)에서 발생하는 모든 프로세스와 이벤트를 모니터링하여 위협을 탐지하고 신속하게 대응합니다.

  • 상황 예제 1: 랜섬웨어 공격 초동 대응

    목표: 직원의 PC에서 랜섬웨어 의심 행위(예: 대량 파일 암호화)가 발생할 경우, 해당 PC를 자동으로 격리하고 공격을 중단시킨다.

    1. EDR 에이전트 설치: 모든 직원 PC에 EDR 에이전트를 설치합니다. 2. 위협 탐지 및 대응 정책 설정: EDR 관리 콘솔에서 "Prevention Policy"를 설정합니다. - Behavioral Analysis: 파일 암호화, 시스템 파일 수정, 의심스러운 외부 통신 등 비정상 행위를 탐지하도록 활성화. - Automated Response: "위협 탐지 시 네트워크 격리" 옵션을 활성화합니다. 3. 결과: A 직원의 PC에서 알 수 없는 프로세스가 대량으로 파일을 암호화하는 행위를 시작하면, EDR이 이를 랜섬웨어로 즉시 탐지하고, 설정된 정책에 따라 A 직원의 PC를 사내 네트워크로부터 자동으로 격리합니다.
  • 상황 예제 2: 악성 스크립트 실행 방지

    목표: 직원이 악성 이메일을 통해 전달받은 `.js` 또는 `.vbs` 스크립트를 실행하는 것을 막는다.

    1. 정책 설정: EDR 관리 콘솔의 "Execution Control" 정책에서 `file_extension`이 `.js`, `.vbs`인 스크립트의 실행을 차단하도록 설정합니다. 2. 결과: 직원이 의도치 않게 악성 스크립트 파일을 더블클릭하더라도, EDR 에이전트가 해당 실행을 즉시 차단하고 관리자에게 알림을 보냅니다.
