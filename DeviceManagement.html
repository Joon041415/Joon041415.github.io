IT 인프라 관리: 기기 관리(Device Management) 가이드

이 가이드는 IT 인프라 관리자 및 실무자를 대상으로, 기업 내에서 사용되는 직원 기기(PC, 노트북, 모바일 장치 등)를 효율적으로 관리하고 보안을 강화하기 위한 실질적인 방법론과 사례를 제공합니다.

1. 기기 관리 관련 서적들의 핵심 주제 리스트

  • 기기 생명 주기 관리 (Device Lifecycle Management): 기기의 도입부터 폐기까지 전 과정을 관리하는 절차.
  • 자산 관리 (Asset Management): 보유 기기의 종류, 수량, 위치, 사용 이력 등을 체계적으로 관리하는 것.
  • 소프트웨어 배포 및 업데이트 관리 (Software Deployment & Update Management): 운영체제 및 애플리케이션의 설치, 패치, 업데이트를 자동화하고 관리하는 것.
  • 보안 관리 (Security Management): 기기 보안 정책 설정, 악성코드 방지, 데이터 보호 등을 포함하는 것.
  • 원격 관리 및 지원 (Remote Management & Support): 사용자의 물리적 위치에 상관없이 기기를 원격으로 제어하고 기술 지원을 제공하는 것.
  • 정책 관리 (Policy Management): 기기 사용에 대한 규칙 및 표준을 설정하고 적용하는 것.

2. 핵심 주제별 설명 및 주요 키워드

기기 생명 주기 관리 (Device Lifecycle Management)

설명: 직원이 새로운 기기를 사용하기 시작하는 시점부터, 사용 기간 동안의 유지보수, 그리고 최종적으로 기기를 폐기하는 시점까지의 모든 과정을 체계화하는 것입니다. 이는 기기의 효율적인 활용과 비용 절감에 직접적인 영향을 미칩니다.
주요 키워드: 도입(Provisioning), 배포(Deployment), 사용 중 관리(In-service Management), 폐기(Decommissioning), 재활용(Recycling).

자산 관리 (Asset Management)

설명: 기업이 소유하거나 임대한 모든 하드웨어 및 소프트웨어 자산의 정보를 정확하게 파악하고 관리하는 활동입니다. 이를 통해 불필요한 구매를 막고, 라이선스 규정을 준수하며, 감사에 대비할 수 있습니다.
주요 키워드: 하드웨어 인벤토리(Hardware Inventory), 소프트웨어 인벤토리(Software Inventory), 라이선스 관리(License Management), 태그(Tagging), 자산 추적(Asset Tracking).

소프트웨어 배포 및 업데이트 관리 (Software Deployment & Update Management)

설명: 새로운 소프트웨어를 모든 직원 기기에 일괄적으로 설치하거나, 기존 소프트웨어의 보안 패치 및 업데이트를 중앙에서 통제하는 과정입니다. 이는 보안 취약점을 줄이고, 모든 사용자가 동일한 최신 버전의 소프트웨어를 사용하도록 보장합니다.
주요 키워드: 패치 관리(Patch Management), 자동화(Automation), 소프트웨어 패키징(Software Packaging), 푸시(Push) 배포, MSI(Microsoft Installer).

보안 관리 (Security Management)

설명: 기기에 저장된 민감한 데이터를 보호하고, 외부 위협으로부터 시스템을 방어하기 위한 모든 관리 활동을 포함합니다. 암호화, 접근 제어, 방화벽 설정 등이 이에 해당합니다.
주요 키워드: 엔드포인트 보안(Endpoint Security), 데이터 암호화(Data Encryption), 디스크 암호화(Disk Encryption), 방화벽(Firewall), 모바일 기기 관리(MDM - Mobile Device Management), 보안 정책(Security Policy).

원격 관리 및 지원 (Remote Management & Support)

설명: 물리적으로 멀리 떨어진 위치에 있는 사용자 기기에 대해 원격으로 문제를 진단하고, 소프트웨어를 설치하거나, 설정을 변경하는 등의 기술 지원을 제공하는 것입니다. 사용자 만족도를 높이고 IT 지원팀의 효율성을 극대화합니다.
주요 키워드: 원격 데스크톱(Remote Desktop), 원격 제어(Remote Control), 실시간 모니터링(Real-time Monitoring), 스크립트 실행(Script Execution), 헬프데스크(Helpdesk).

정책 관리 (Policy Management)

설명: 조직의 보안 및 운영 표준에 따라 기기 사용에 대한 규칙을 설정하고 모든 기기에 일관적으로 적용하는 것입니다. 예를 들어, USB 포트 사용 금지, 특정 애플리케이션 설치 제한 등이 있습니다.
주요 키워드: 그룹 정책(Group Policy), 디바이스 제어(Device Control), 구성 프로필(Configuration Profile), 접근 제어(Access Control).

3. 기기 관리에서 주요 사용 도구의 상황별 구체적인 사용 예제

가. Microsoft Endpoint Configuration Manager (MECM, 구 SCCM)

MECM은 대규모 Windows 기반 환경에서 기기 관리를 위한 강력한 엔터프라이즈 솔루션입니다.

  • 상황 예제 1: 신규 입사자 PC 자동화 설정

    목표: 신규 입사자에게 지급될 PC에 필수 소프트웨어(Office 365, 사내 메신저, VPN 클라이언트 등)를 자동으로 설치하고 보안 정책을 적용한다.

    사용 예제:

    1. 소프트웨어 패키징: 각 소프트웨어의 설치 파일을 .msi 또는 .exe 형태로 패키징하여 MECM의 "Software Library"에 등록합니다. 2. 컬렉션(Collection) 생성: 신규 입사자들을 위한 Active Directory 그룹(`New_Employees`)을 기반으로 MECM에 "신규 입사자 PC" 컬렉션을 생성합니다. 3. 배포(Deployment) 설정: Office 365 패키지를 `New_Employees` 컬렉션에 "필수(Required)" 배포로 설정합니다. "배포 일정"을 설정하여 사용자가 PC를 켜는 즉시 설치가 시작되도록 합니다. 4. 정책 적용: "Configuration Items" 기능을 사용하여 바탕화면 이미지, 비밀번호 복잡성 요구 사항 등의 보안 정책을 설정하고, "Configuration Baselines"를 통해 해당 컬렉션에 적용합니다.
  • 상황 예제 2: 전사적 보안 패치 배포

    목표: 윈도우 운영체제 및 주요 애플리케이션(Chrome, Acrobat Reader)의 최신 보안 패치를 모든 기기에 일괄 적용한다.

    사용 예제:

    1. 동기화: MECM의 "Software Update Point"를 통해 Microsoft Update와 동기화하여 최신 패치 목록을 가져옵니다. 2. 소프트웨어 업데이트 그룹 생성: 이번 달에 배포할 모든 윈도우 보안 패치들을 그룹화하여 `2025_08_Security_Patches`라는 그룹을 생성합니다. 3. 배포: `2025_08_Security_Patches` 그룹을 "모든 데스크톱 및 서버" 컬렉션에 배포합니다. 배포 옵션에서 사용자에게 "새로운 업데이트가 있습니다. 지금 설치하시겠습니까?"라는 알림을 띄우고, 특정 시간(예: 금요일 업무 종료 후)에 자동 재부팅되도록 설정합니다.

나. Intune (Microsoft Endpoint Manager)

Intune은 클라우드 기반의 통합 엔드포인트 관리 솔루션으로, 특히 모바일 기기(MDM)와 클라우드 기반 PC 관리에 특화되어 있습니다.

  • 상황 예제 1: 모바일 기기(BYOD) 보안 정책 적용

    목표: 직원 개인 소유의 모바일 기기(BYOD)가 회사 메일에 접근할 때, 최소한의 보안 정책(비밀번호 설정, 암호화 등)을 준수하도록 강제한다.

    사용 예제:

    1. 정책 생성: Intune 관리 센터에서 "장치" > "구성 프로필"로 이동하여 새로운 iOS/Android용 구성 프로필을 생성합니다. 2. 설정: "장치 기능 제한" 정책에서 "암호"를 "요구"로 설정하고, 비밀번호 복잡성을 "영숫자"로, 최소 길이를 "8자리"로 지정합니다. "장치 암호화"도 "요구"로 설정합니다. 3. 할당: 해당 정책을 회사 메일(`Corporate_Email`) 접근을 허용할 "모든 BYOD 사용자" 그룹에 할당합니다. 4. 결과: 사용자가 회사 메일 앱을 설치하면, Intune이 자동으로 정책을 적용하며, 정책을 준수하지 않을 경우 메일 접근을 차단합니다.
  • 상황 예제 2: 특정 앱의 설치 허용/차단

    목표: 직원 PC에 보안상 문제가 될 수 있는 특정 앱(`TeamViewer`)의 설치를 금지하고, 필수 앱(`Slack`)만 설치 가능하게 한다.

    사용 예제:

    1. 앱 차단: Intune 관리 센터에서 "앱" > "앱 구성 정책"으로 이동합니다. 2. 정책 생성: Windows 10/11용 정책을 생성하고, "블랙리스트 앱" 목록에 `TeamViewer.exe`를 추가합니다. 이 정책을 "모든 직원 PC" 그룹에 할당합니다. 3. 앱 배포: "앱" > "모든 앱"에서 "Microsoft Store 앱"을 선택하여 `Slack` 앱을 추가합니다. 이 앱을 "필수 배포"로 설정하여 모든 직원 PC에 자동으로 설치되도록 합니다. // 주석: `TeamViewer`와 같은 원격 제어 앱은 보안 위험이 높아 많은 기업에서 금지하는 경우가 많습니다. `TeamViewer.exe` 대신 앱의 해시 값이나 기타 식별자를 사용하여 더 정확하게 차단할 수 있습니다.

다. Group Policy (그룹 정책)

Group Policy는 Windows Server Active Directory(AD) 환경에서 PC 설정을 중앙에서 관리하는 가장 기본적인 도구입니다.

  • 상황 예제 1: 모든 PC에 동일한 보안 설정 적용

    목표: 모든 PC의 USB 저장 장치 사용을 금지하고, 특정 웹사이트(`www.games.com`) 접근을 차단한다.

    사용 예제:

    1. GPO(Group Policy Object) 생성: Windows Server의 "그룹 정책 관리" 콘솔에서 새로운 GPO를 생성합니다. GPO 이름을 `PC_Security_Policy`로 지정합니다. 2. 정책 편집: `PC_Security_Policy` GPO를 마우스 오른쪽 버튼으로 클릭하여 "편집"을 선택합니다. 3. USB 차단: `컴퓨터 구성 > 관리 템플릿 > 시스템 > 이동식 저장소 액세스`로 이동합니다. "모든 이동식 저장소 클래스: 모든 액세스 거부" 정책을 "사용"으로 설정합니다. 4. 웹사이트 차단: `컴퓨터 구성 > 관리 템플릿 > Windows 구성 요소 > Internet Explorer`로 이동합니다. `콘텐츠 필터링` 정책을 설정하여 `www.games.com`을 차단 목록에 추가합니다. 5. GPO 연결: `PC_Security_Policy` GPO를 모든 PC가 포함된 `Domain Computers` OU(조직 구성 단위)에 연결합니다.
  • 상황 예제 2: 사용자 계정별로 다른 설정 적용

    목표: 일반 직원 계정에는 제어판 접근을 막고, IT 관리자 계정에는 모든 권한을 허용한다.

    사용 예제:

    1. OU 구성: Active Directory에서 `General_Users`와 `IT_Admins` OU를 생성하고 각 계정을 해당 OU에 배치합니다. 2. GPO 생성 및 연결: `General_Users_Policy` GPO를 `General_Users` OU에, `IT_Admins_Policy` GPO를 `IT_Admins` OU에 각각 연결합니다. 3. 정책 편집: `General_Users_Policy` GPO를 편집하여 `사용자 구성 > 관리 템플릿 > 제어판`에서 "제어판 액세스 금지" 정책을 "사용"으로 설정합니다. `IT_Admins_Policy`에는 해당 설정을 하지 않습니다. 4. 결과: `General_Users` OU에 속한 사용자가 PC에 로그인하면 제어판 접근이 자동으로 차단됩니다.
